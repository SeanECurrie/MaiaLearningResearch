# 04-QA-DOCUMENTATION

## Purpose
Quality assurance documentation, reports, and change logs.

## Contents
- `qa-reports/` - QA review reports for each version
- `change-logs/` - Detailed logs of changes made during QA
- `verification/` - Verification checklists and sign-offs

## When to Use
- After completing deliverables (before client delivery)
- When tracking QA findings and fixes
- When documenting changes between versions
- For audit trail and quality verification

## QA Process
1. Run `competitive-analysis-quality-assurance` skill on deliverables
2. Document findings in `qa-reports/`
3. Fix issues and document in `change-logs/`
4. Verify fixes and update `verification/`

## Related Directories
- `03-DELIVERABLES/` - Deliverables being QA'd
- `05-ITERATIONS/` - Iteration-specific documentation
- `templates/competitive-analysis/04-QA-PHASE.md` - QA methodology guide
